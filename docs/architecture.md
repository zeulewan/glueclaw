# Architecture

## Flow

1. Registers provider `glueclaw` with three models (opus, sonnet, haiku)
2. Spawns `claude -p --output-format stream-json`
3. Scrubs [detection triggers](detection-patterns.md) from the system prompt
4. Translates NDJSON events into OpenClaw's AssistantMessageEventStream
5. Bridges OpenClaw gateway tools via MCP loopback
6. Persists sessions to `~/.glueclaw/sessions.json` for multi-turn resume
7. Translates response tokens back to gateway format

## Source files

| File | Purpose |
|------|---------|
| `index.ts` | Plugin registration, model catalog, stream function wiring |
| `src/stream.ts` | Subprocess, NDJSON parsing, scrub chain, sessions, MCP bridge |
| `install.sh` | Installer: deps, plugin registration, model config, gateway patch |
| `openclaw.plugin.json` | Plugin manifest |

## Scrub chain

`scrubPrompt()` in `src/stream.ts` replaces detection triggers before passing the system prompt to the CLI:

```
OpenClaw identity phrase  → GlueClaw
HEARTBEAT_OK              → GLUECLAW_ACK
reply_to_current          → reply_current
[[reply_to:               → [[reply:
openclaw.inbound_meta     → glueclaw.inbound_meta
generated by OpenClaw     → generated by GlueClaw
```

`unscrubResponse()` reverses the first three translations in response text.

## Sessions

- Stored in `~/.glueclaw/sessions.json`
- CLI runs from `cwd: ~/.glueclaw`
- Resume uses `--resume <session_id>`, skips `--system-prompt`

## MCP bridge

The installer patches `server-*.js` in the OpenClaw dist to expose `__GLUECLAW_MCP_PORT` and `__GLUECLAW_MCP_TOKEN` via `process.env`. The stream function reads these, writes a temporary MCP config, and passes `--strict-mcp-config --mcp-config <file>` to the subprocess.
