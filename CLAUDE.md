# GlueClaw

OpenClaw provider plugin that routes inference through the real `claude` CLI binary, using your Claude Max subscription instead of API keys.

## How it works

1. Registers as provider `glueclaw` with models: glueclaw-opus, glueclaw-sonnet, glueclaw-haiku
2. `createStreamFn` spawns `claude -p --output-format stream-json` as a subprocess
3. Scrubs Anthropic detection triggers from the system prompt (see DETECTION_PATTERNS.md)
4. Translates NDJSON stream events into OpenClaw's AssistantMessageEventStream format
5. OpenClaw gateway tools bridged via MCP loopback server
6. Sessions persist to `~/.glueclaw/sessions.json` and resume across gateway restarts via `--resume`
7. Response tokens (GLUECLAW_ACK, reply_current) translated back to gateway-expected format

## Files

- `index.ts` - plugin registration, model catalog, stream function wiring
- `src/stream.ts` - core: subprocess spawning, NDJSON parsing, MCP bridge, session persistence, token translation
- `install.sh` - automated installer: deps, plugin registration, model config, auth, gateway MCP patch
- `get.sh` - curl bootstrap (clones repo then runs install.sh)
- `openclaw.plugin.json` - plugin manifest
- `DETECTION_PATTERNS.md` - documented Anthropic triggers and binary search procedure
- `HEALTHCHECK.md` - daily monitoring prompt and procedure
- `TESTS.md` - test procedures for all features
- `TODO.md` - known issues and future work

## Scrub chain (stream.ts)

```typescript
.replace(/personal assistant running inside OpenClaw/g, "...GlueClaw")
.replace(/HEARTBEAT_OK/g, "GLUECLAW_ACK")
.replace(/reply_to_current/g, "reply_current")
.replace(/\[\[reply_to:/g, "[[reply:")
.replace(/openclaw\.inbound_meta/g, "glueclaw.inbound_meta")
.replace(/generated by OpenClaw/g, "generated by GlueClaw")
```

## Translation layer (response -> gateway)

```typescript
text.replace(/GLUECLAW_ACK/g, "HEARTBEAT_OK")
text.replace(/reply_current/g, "reply_to_current")
text.replace(/\[\[reply:/g, "[[reply_to:")
```

Applied in both streaming deltas and endStream.

## Session persistence

- Sessions saved to `~/.glueclaw/sessions.json` (survives gateway restarts)
- Claude subprocess runs from `cwd: ~/.glueclaw` so Claude Code stores session data there
- On resume, `--system-prompt` is NOT passed (session already has it)
- New sessions get the scrubbed prompt via `--system-prompt`

## MCP bridge

- Gateway patches `server*.js` to expose loopback port/token via `process.env.__GLUECLAW_MCP_PORT/TOKEN`
- Stream function reads port/token, writes temp MCP config, passes `--strict-mcp-config --mcp-config <file>`
- 19 OpenClaw tools available including message, cron, sessions, memory, browser, web search

## Install

The install script:
1. npm install for pi-ai/pi-agent-core deps
2. Adds GLUECLAW_KEY=local to shell profile
3. Registers plugin (with fallback for older OpenClaw versions)
4. Configures models and gateway.tools.allow
5. Writes auth profile
6. Patches gateway dist for MCP token exposure
7. Starts gateway

## Uninstall

Switch model and restore patched file:
```bash
openclaw config set agents.defaults.model anthropic/claude-sonnet-4-6
cd "$(dirname "$(which openclaw)")/../lib/node_modules/openclaw/dist" && for f in *.glueclaw-bak; do [ -f "$f" ] && mv "$f" "${f%.glueclaw-bak}"; done
openclaw gateway restart
```
